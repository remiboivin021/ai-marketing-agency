# Prompting Guidelines

This file explains how to frame requests so AI outputs are constrained, reviewable, and useful.

## Include in prompts

- Goal: what must be achieved.
- Scope: which files or areas may change.
- Constraints: architecture, security, compatibility, style.
- Validation: how success will be checked.
- Non-goals: what must not be changed.

## Good prompt shape

Use the form: context, task, constraints, acceptance criteria.

## Avoid

- Ambiguous success criteria.
- Requests that mix unrelated tasks.
- Asking for speculative features not grounded in repository context.
