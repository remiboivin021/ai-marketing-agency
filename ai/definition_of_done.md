# Definition Of Done

This file defines when AI-assisted work is considered complete enough for human review.

## Minimum done criteria

- The requested scope is implemented or documented.
- No known contract break is introduced silently.
- Edge cases relevant to the change are considered.
- Tests are added or the testing gap is stated explicitly.
- Documentation is updated when behavior, config, or interfaces change.
- Risks and assumptions are visible to the reviewer.

## Not done

- "Seems to work" without verification.
- Hidden TODOs without explicit approval.
- Partial changes presented as complete.
