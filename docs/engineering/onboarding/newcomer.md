# Newcomer Guide

This document helps a new contributor get productive without relying on implicit project knowledge.

## Start here

1. Read `PROJECT.md` for project overview
2. Read `AGENTS.md` if AI assistance is involved
3. Read the relevant `docs/` domain index
4. Identify the local validation path before changing anything

## First-day checklist

- Understand scope of the project
- Locate the main entry points (code, tests, configuration)
- Confirm how changes are reviewed and merged
- Set up local development environment
- Run the build locally to verify it works
- Read the coding standards
