# Governance

This directory defines **how decisions are made and recorded**.

It exists to make authority and responsibility explicit.

---

## Purpose

Governance prevents:
- hidden decisions
- unclear ownership
- inconsistent direction

---

## Contents

- `decision-process.md`  
  How decisions are proposed, discussed, and accepted.

- `contribution-model.md`  
  Roles, responsibilities, and contribution rules.

- `adr/`  
  Architecture Decision Records.

---

## Rules

- Significant decisions must be documented
- ADRs are required for architectural changes
- Process should remain lightweight and explicit
