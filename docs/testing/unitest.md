# Unit Tests

This document defines what belongs in unit-level validation and what does not.

## Cover with unit tests

- Deterministic business logic
- Input validation
- Branching behavior
- Error handling with stable expectations

## Avoid

Do not treat implementation-coupled tests as a substitute for behavioral coverage.
