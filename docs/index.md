# Documentation Index

This documentation is structured to support different project maturity levels.

You do not need to use everything.

SynthKit is modular by design.

---

# Choose your path

## 1️⃣ Quick Start (Makers / Solo Dev)

If you want to start fast, focus only on:

- `PROJECT.md`
- `context/product_context.md`
- `ai/coding_rules.md`
- `docs/architecture/index.md`

That is enough to build a structured AI-assisted project.

Ignore the rest for now.

---

## 2️⃣ Team Mode (Growing Project)

If you are working with multiple contributors, also use:

- `docs/governance/`
- `docs/tooling/`
- `docs/engineering/`
- `docs/testing/`

This ensures alignment, reproducibility, and quality control.

---

## 3️⃣ Structured / Regulated Mode

If your domain requires traceability or compliance, use:

- `docs/requirements/`
- `docs/requirements/traceability_matrix.md`
- `docs/architecture/security-architecture.md`
- `docs/architecture/deployment.md`

These sections are optional unless your project requires formal validation.

# Guiding Principle

Documentation should serve clarity.

If a document does not improve clarity, remove or simplify it.

SynthKit favors explicit structure over bureaucratic volume.
