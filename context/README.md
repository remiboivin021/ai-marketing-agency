# Project Context

This directory contains the **core contextual knowledge** of the project.

It is the primary input for both humans and AI.

---

## Purpose

Context prevents incorrect optimization.

Before writing code or generating outputs, this directory must be read and understood.

---

## Contents

- `product_context.md`  
  What the product is, who it is for, and what problem it solves.

- `technical_context.md`  
  Technical environment, stack, constraints, and assumptions.

- `constraints.md`  
  Non-negotiable constraints (performance, security, cost, compliance).

---

## Rules

- Keep documents explicit and up to date
- Avoid vague statements
- Prefer clarity over completeness

If context is missing or outdated, fix it before coding.
